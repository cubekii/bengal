Introduction

Bengal is a modern, expressive programming language designed with simplicity and power in mind. It features a clean syntax, strong typing, and built-in support for modern programming paradigms including object-oriented programming, async/await, and reflection.

Tip: Bengal uses the .bl file extension for source files.

Why Bengal?

  • Clean Syntax: Easy to read and write, with minimal boilerplate
  • Modern Features: Async/await, classes, modules, and more
  • Reflection: Built-in runtime type inspection
  • JSON Support: Native JSON parsing and serialization
  • HTTP Client: Built-in HTTP capabilities
  • FFI Support: Foreign Function Interface for C interoperability

Getting Started

Installation

To get started with Bengal, clone the repository and build from source:

Terminal 
git clone https://github.com/cubek/bengal.git
cd bengal
cargo build --release

Your First Program

Create a file named hello.bl:

hello.bl 
import std::io

println("Hello, World!")

Run it with the Bengal compiler:

Terminal 
bengal run hello.bl

Basic Syntax

Comments

Bengal supports both line comments and block comments:

comments.bl 
// This is a line comment

/* This is a block comment */

/*
 * Multi-line block comment
 * with multiple lines
 */

let x = 10  // inline comment
let y = 20  /* inline block comment */

String Interpolation

Use ${} syntax to embed expressions in strings:

interpolation.bl 
let name = "Bengal"
let version = 1.0

println("Welcome to ${name} version ${version}!")
// Output: Welcome to Bengal version 1.0!

Multiline Strings

Use triple quotes for multiline strings with automatic indentation stripping:

multiline.bl 
let text = """
    This is a multiline string.
    Indentation is automatically stripped.
    Variables work: ${name}
"""

println(text)

Variables & Types

Variable Declaration

Use let to declare mutable variables:

variables.bl 
let x = 42
let message = "Hello"

Basic Types

Type Description Example
int 64-bit integer (i64) let x: int = 42
float 64-bit float (f64) let pi: float = 3.14
str String let s: str = "hello"
bool Boolean let b: bool = true

Type Annotations

Types can be explicitly annotated or inferred:

types.bl 
// Type inference
let x = 42        // inferred as int
let pi = 3.14     // inferred as float

// Explicit types
let y: int = 10
let a: float = 2.0

Operators

Bengal supports standard arithmetic and comparison operators:

operators.bl 
// Arithmetic
let sum = x + y
let diff = x - y
let product = x * y
let quotient = x / y

// Comparison
if (x == 42) { }
if (x != 0) { }
if (x > y) { }
if (x < y) { }

Functions

Function Declaration

Use fn to declare functions with optional type annotations:

functions.bl 
fn sum(int x, y): int {
  return x + y
}

fn mul(int x, int y): int {
  return x * y
}

// Calling functions
let result = sum(10, 20)
println("Sum: ${result}")

Function Parameters

Parameters can have types specified once for multiple parameters:

params.bl 
// Both x and y are int
fn add(int x, y): int {
    return x + y
}

Control Flow

If/Else

if.bl 
if (x > 10) {
    println("x is greater than 10")
} else if (x == 10) {
    println("x equals 10")
} else {
    println("x is less than 10")
}

For Loops

Use for...in for range-based loops:

for.bl 
// Count up: 1, 2, 3, 4
for (i in 1..5) {
    println(i)
}

// Count down: 10, 9, 8, 7, 6
for (i in 10..5) {
    println(i)
}

While Loops

while.bl 
while (condition) {
    println("Loop!")
    // update condition
}

Classes & OOP

Bengal supports object-oriented programming with classes, methods, and access modifiers.

Class Definition

class.bl 
class SomeObject {
  some_int: int = 10
  some_float: float = 5.0
  some_string: str = "Default string"

  private some_private: bool = false

  fn someMethod(): str? {
    if (!some_private) {
      some_private = true
      return null
    }
    return some_string
  }
}

// Creating instances
let some_object = SomeObject()
println("Call 1: ${some_object.someMethod()}")
println("Call 2: ${some_object.someMethod()}")

Access Modifiers

  • public (default): Accessible from anywhere
  • private: Accessible only within the class

Nullable Types

Use ? to indicate a type can be null:

nullable.bl 
fn findUser(int id): str? {
    // May return null
    return null
}

Async/Await

Bengal supports asynchronous programming with async and await keywords.

async.bl 
async fn fetch_example(): str {
  // Make async HTTP request
  // let response = await http.get("https://api.example.com/data")
  return "HTTP async is implemented!"
}

// Using async functions
// let result = await fetch_example()
Note: Async functions can only be called with await from within other async functions.

Error Handling

Bengal uses try-catch blocks for error handling.

Try-Catch

try-catch.bl 
try {
    println("Entering try block")
    throw "Exception!"
    println("This should not be reached")
} catch (e) {
    println("Caught: " + e)
}

println("Continuing after try-catch")

Nested Try-Catch

nested-try.bl 
try {
    println("Outer try")
    try {
        println("Inner try")
        throw "Inner Exception"
    } catch (e) {
        println("Caught in inner: " + e)
        throw "Re-thrown: " + e
    }
} catch (e) {
    println("Caught in outer: " + e)
}

Advanced Features

Reflection

Bengal has built-in reflection capabilities:

reflection.bl 
import std::io

class User {
  name: str = "Alice"
  age: int = 30
}

let u = User()

// Get type information
let type = std::reflect::type_of(u)
let className = std::reflect::class_name(u)

// Get fields as object
let fields = std::reflect::fields(u)
println("fields.name: ${fields.name}")

JSON Support

json.bl 
import std::json

// Stringify object to JSON
let jsonStr = std::json::stringify(u)
println("JSON: ${jsonStr}")

// Parse JSON string
let parsed = std::json::parse(jsonStr)
println("Parsed name: ${parsed.name}")

HTTP Client

http.bl 
import std::http

// Using HttpMethod enum
let method = HttpMethod.GET

// Using RedirectPolicy enum
let policy = RedirectPolicy.Follow

// Create HTTP client
let client = HttpClient.new()
client.set_timeout(5000)
client.set_verify_ssl(true)

// Add headers
client.add_header("User-Agent", "Bengal/1.0")
client.add_header("Accept", "application/json")

Environment Variables

env.bl 
import std::sys

// Get environment variable
println("PATH: " + std::sys::env("PATH"))
