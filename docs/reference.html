Lexical Structure

Identifiers

Identifiers in Bengal start with a letter or underscore, followed by letters, digits, or underscores:

Pattern Description Examples
[a-zA-Z_][a-zA-Z0-9_]* Valid identifier name, _private, myVar123

Keywords

The following keywords are reserved in Bengal:

async     await     class     const     else
fn        for       if        import    in
let       module    native    null      private
return    str       throw     try       while
int       float     bool      void      any

Literals

Type Syntax Examples
Integer [0-9]+ 42, 0, -10
Float [0-9]+\.[0-9]+ 3.14, 2.0, -0.5
String "..." or """...""" "hello", """multiline"""
Boolean true or false true, false
Null null null

Comments

Type Syntax Example
Line comment // ... // This is a comment
Block comment /* ... */ /* Multi-line comment */

Types

Primitive Types

Type Size Description Range
int 64-bit Signed integer (i64) -2⁶³ to 2⁶³-1
float 64-bit IEEE 754 float (f64) ±5.0×10⁻³²⁴ to ±1.7×10³⁰⁸
str variable UTF-8 string limited by memory
bool 8-bit Boolean true or false

Nullable Types

Any type can be made nullable by appending ?:

fn findUser(int id): str? {
    return null  // Can return null
}

let maybeValue: int? = null

Type Inference

Bengal uses type inference when explicit types are not provided:

let x = 42       // x: int
let pi = 3.14    // pi: float
let s = "hi"     // s: str

String Interpolation

let name = "Bengal"
let version = 1.0

// ${expression} syntax
println("Welcome to ${name} v${version}!")

Range Expressions

Ranges use the .. operator for iteration:

// Ascending range: 1, 2, 3, 4
for (i in 1..5) { }

// Descending range: 10, 9, 8, 7, 6
for (i in 10..5) { }

Statements

Variable Declaration

Syntax Description
let name = value Mutable variable with type inference
let name: Type = value Mutable variable with explicit type

If Statement

if (condition) {
    // executed if condition is true
} else if (otherCondition) {
    // executed if otherCondition is true
} else {
    // executed if all conditions are false
}

For Loop

for (variable in start..end) {
    // loop body
}

While Loop

while (condition) {
    // loop body
}

Try-Catch

try {
    // code that may throw
    throw "error message"
} catch (e) {
    // error handling
    println("Error: " + e)
}

Functions

Function Declaration Syntax

fn functionName(param1: Type, param2: Type): ReturnType {
    // function body
    return value
}

Parameter Type Shorthand

When consecutive parameters have the same type, specify it once:

// Both x and y are int
fn add(int x, y): int {
    return x + y
}

Return Types

Syntax Description Example
: int Returns integer fn getNum(): int
: str? Returns nullable string fn find(): str?
(no return type) Returns void fn print()

Async Functions

async fn fetchData(url: str): str {
    let response = await http.get(url)
    return response
}

Classes

Class Declaration

class ClassName {
    // Fields with default values
    fieldName: Type = defaultValue
    
    // Private field
    private secret: str = "hidden"
    
    // Methods
    fn methodName(): ReturnType {
        return value
    }
}

Access Modifiers

Modifier Visibility
(none) Public - accessible from anywhere
private Private - accessible only within the class

Constructor

Classes are instantiated by calling the class name as a function:

class Point {
    x: int = 0
    y: int = 0
}

let p = Point()  // Creates instance with default values

Methods

class Calculator {
    fn add(int x, y): int {
        return x + y
    }
    
    fn multiply(int x, y): int {
        return x * y
    }
}

let calc = Calculator()
let result = calc.add(5, 3)

Modules

Module Declaration

Modules are declared using the module keyword with :: namespace separator:

module std::io
module myapp::utils
module myapp::models::user

Import Statement

// Import entire module
import std::io

// Import specific items (future feature)
// import std::io::{println, print}

Module Path Resolution

Path Description
std::io Standard library io module
std::http Standard library http module
std::json Standard library json module
std::reflect Standard library reflection module
std::sys Standard library system module

Patterns

For-In Pattern

The for...in pattern iterates over ranges:

for (i in 0..10) {
    println(i)
}

Range Behavior

Range Behavior Values
1..5 Ascending (start < end) 1, 2, 3, 4
10..5 Descending (start > end) 10, 9, 8, 7, 6

Grammar Summary

Program Structure

program     ::= (moduleDecl)? (import)* (topLevel)*
moduleDecl  ::= "module" IDENTIFIER ("::" IDENTIFIER)*
import      ::= "import" IDENTIFIER ("::" IDENTIFIER)+
topLevel    ::= function | class | statement

Function Grammar

function    ::= ("async")? "fn" IDENTIFIER "(" params? ")" (":" type)? block
params      ::= param ("," param)*
param       ::= type? IDENTIFIER
type        ::= ("int" | "float" | "str" | "bool" | "void") ("?")?

Class Grammar

class       ::= "class" IDENTIFIER "{" (field | method)* "}"
field       ::= ("private")? IDENTIFIER ":" type ("=" expression)?
method      ::= function

Expression Grammar

expression  ::= literal | identifier | binaryOp | unaryOp | call | access
literal     ::= INT | FLOAT | STRING | "true" | "false" | "null"
binaryOp    ::= expression ("+" | "-" | "*" | "/" | "==" | "!=" | ">" | "<") expression
unaryOp     ::= ("!" | "-") expression
call        ::= IDENTIFIER "(" arguments? ")"
access      ::= expression "." IDENTIFIER

Statement Grammar

statement   ::= letStmt | ifStmt | forStmt | whileStmt | tryStmt | returnStmt | exprStmt
letStmt     ::= "let" IDENTIFIER (":" type)? "=" expression
ifStmt      ::= "if" "(" expression ")" block ("else" ifStmt | "else" block)?
forStmt     ::= "for" "(" "let" IDENTIFIER "in" expression ".." expression ")" block
whileStmt   ::= "while" "(" expression ")" block
tryStmt     ::= "try" block "catch" "(" IDENTIFIER ")" block
returnStmt  ::= "return" expression?
